\section{Deployment Diagram}

The deployment diagram presents the physical architecture of the system and shows how software components are deployed across hardware nodes \cite{booch2005uml,owler2010uml,UML251OMG2017}.

It illustrates:
\begin{itemize}
	\item Client devices used to access the system (web and mobile users)
	\item Application server hosting the business logic
	\item Database server responsible for data storage
	\item Communication protocols enabling interaction between components
\end{itemize}

This diagram provides a clearer understanding of how the system is executed in a real environment and how its different components cooperate to deliver the required services. The modeling notation and structure were inspired by the UML 2.5.1 specification \cite[p.~653]{UML251OMG2017}.
